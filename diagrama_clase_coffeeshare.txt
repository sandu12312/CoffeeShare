INSTRUCȚIUNI PENTRU BOXURI ALBASTRE:
====================================

1. Copiază codul de mai sus COMPLET (cu configurația de tema)
2. Deschide Mermaid Live Editor: https://mermaid.live/
3. Lipește codul în editor
4. Boxurile ar trebui să apară albastre datorită configurației de tema
5. Dacă nu funcționează, încearcă să schimbi tema din setări la "base"

ALTERNATIV - CSS CUSTOM:
========================

Dacă boxurile nu sunt albastre, adaugă acest CSS în Mermaid Live Editor:

```css
.node rect {
    fill: #BBDEFB !important;
    stroke: #1976D2 !important;
    stroke-width: 2px !important;
}

.node text {
    fill: #000000 !important;
}
```

SIMBOLURI UML FOLOSITE:
=======================

- *-- = Composition (diamant plin negru)
- o-- = Aggregation (diamant gol)  
- -- = Association (linie simplă)
- <|-- = Inheritance (triunghi gol)
- ..> = Dependency (linie punctată cu săgeată)

RELAȚIILE IMPLEMENTATE:
=======================

COMPOSITION (diamant plin):
- User *-- Subscription (utilizatorul deține abonamentele)
- User *-- QRCode (utilizatorul generează codurile QR)
- Subscription *-- Transaction (abonamentul conține tranzacțiile)

AGGREGATION (diamant gol):
- User o-- SecurityService (utilizatorul are serviciu de securitate)

ASSOCIATION (linie simplă):
- User -- Analytics (utilizatorul este urmărit de analytics)
- QRCode -- Transaction (codul QR este folosit în tranzacție)
- CoffeeShop -- Transaction (cafeneaua validează tranzacții)
- CoffeeShop -- User (cafeneaua aparține utilizatorului)

INHERITANCE (triunghi):
- User <|-- Admin (admin extinde utilizatorul)

DEPENDENCY (linie punctată):
- Subscription ..> PaymentService (abonamentul folosește serviciul de plată)
- QRCode ..> SecurityService (codul QR este validat de serviciul de securitate)
- Transaction ..> Analytics (tranzacția alimentează cu date analytics-ul)
